# Watch this before you build a GHL marketplace app

**Source:** https://www.youtube.com/watch?v=huevGw_mgYA
**Author:** Adam Skjervold

Here are the key points from the GoHighLevel tutorial video:

**Key Features:**
* Workflows
* Webhooks
* Custom actions
* Custom fields
* Custom objects
* Database structure

**Step-by-Step Setup Instructions:**
* Set up a separate database (e.g. Supabase, Firebase, Postgress) to act as a single source of truth
* Integrate the database with GoHighLevel and other apps (e.g. Square)
* Use the database to store and manage customer data, including phone numbers, emails, and other profiles
* Use the database to keep track of multiple data points and add layers of organization

**API/Automation Capabilities:**
* Use webhooks to automate workflows
* Use custom actions to automate tasks
* Integrate with other apps (e.g. Zapier, n8n) to automate workflows
* Use a separate database to automate data management and synchronization

**Potential Marketplace App Ideas:**
* Custom database integration apps
* Automation workflow apps
* Data synchronization apps
* Customer profile management apps
* E-commerce store integration apps
* Booking system integration apps
* Custom functionality apps

**Best Practices:**
* Don't try to keep everything within GoHighLevel
* Use a separate database as a single source of truth
* Don't try to use funnels for everything
* Keep your database organized and add layers of organization as needed
* Use automation and API capabilities to streamline workflows and data management

**Pro Tips:**
* Have a single source of truth built into your system
* Don't try to make funnels work for everything
* Use a separate database to manage customer data and automate workflows.
