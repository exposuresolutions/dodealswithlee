Check Your System

Open PowerShell and run:

# Check Windows version
winver

# Check RAM
systeminfo | findstr "Total Physical Memory"

# Check disk space

2. Windows 10 Optimization for AI Workloads

Old mini PCs need every bit of performance. Run these optimizations before installing AI tools.

Step 2.1 — Disable Startup Bloat

# Open Task Manager > Startup tab
# Disable everything except:
#   - Windows Security
#   - Audio drivers (if needed)
# Right-click each item > Disable

Step 2.2 — Disable Unnecessary Services

# Open PowerShell as Administrator
# Disable telemetry and tracking
Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows\DataCollection" -Name "AllowTelemetry" -Value 0

# Disable Windows Search indexing (saves CPU + disk)
Stop-Service WSearch
Set-Service WSearch -StartupType Disabled

# Disable Superfetch/SysMain (helps on slow SSDs/HDDs)
Stop-Service SysMain
Set-Service SysMain -StartupType Disabled

Step 2.3 — Set Power Plan to High Performance

powercfg /setactive 8c5e7fda-e8bf-4a96-9a85-a6e23a8c635c

Step 2.4 — Clean Up Disk Space

# Run Disk Cleanup
cleanmgr /d C

# Clear temp files
Remove-Item "$env:TEMP\*" -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item "C:\Windows\Temp\*" -Recurse -Force -ErrorAction SilentlyContinue

# Clear Windows Update cache (if space is critical)
Stop-Service wuauserv
Remove-Item "C:\Windows\SoftwareDistribution\Download\*" -Recurse -Force
Start-Service wuauserv

Step 2.5 — Virtual Memory (Swap)

If RAM is 8 GB, increase the page file:

  1. System Properties > Advanced > Performance Settings > Advanced > Virtual Memory
  2. Uncheck "Automatically manage"
  3. Set Custom Size: Initial = 8192 MB, Maximum = 16384 MB
  4. Click Set > OK > Restart
White-Label Note: For client deployments, create a PowerShell script that runs all optimization steps automatically. Save as optimize-ai-workstation.ps1 in the client's setup folder.

3. Security Hardening

Security Philosophy: Lock down the system, but give the AI agent a controlled sandbox to work in. The agent can learn and do serious work within its boundaries. It cannot touch system files, other users' data, or make unauthorized network requests.

Step 3.1 — Create a Dedicated AI User Account

# PowerShell as Administrator
# Create a standard (non-admin) user for the AI workstation
net user AIAgent StrongPassword123! /add
# Do NOT add to Administrators group

# Create a workspace folder the AI user owns
mkdir C:\AIWorkspace
icacls C:\AIWorkspace /grant AIAgent:(OI)(CI)F
Why: The AI agent runs under a standard user account. It can read/write its workspace but cannot install software, modify system settings, or access other users' files.

Step 3.2 — Windows Firewall Rules

# Allow outbound HTTPS (APIs need this)
New-NetFirewallRule -DisplayName "AI-Allow-HTTPS" -Direction Outbound -Protocol TCP -RemotePort 443 -Action Allow -Profile Any

# Allow outbound to specific AI API domains (optional, stricter)
# Claude: api.anthropic.com
# ElevenLabs: api.elevenlabs.io
# Google: generativelanguage.googleapis.com
# Groq: api.groq.com
# GHL: services.leadconnectorhq.com

# Block all other outbound from AI workspace (optional, very strict)
# New-NetFirewallRule -DisplayName "AI-Block-Other" -Direction Outbound -Action Block -Program "C:\AIWorkspace\*"

Step 3.3 — Windows Defender Configuration

# Ensure Defender is active
Set-MpPreference -DisableRealtimeMonitoring $false

# Add AI workspace to monitored folders
Add-MpPreference -ControlledFolderAccessAllowedApplications "C:\AIWorkspace\python.exe"

# Enable controlled folder access (ransomware protection)
Set-MpPreference -EnableControlledFolderAccess Enabled

Step 3.4 — Environment Variable Security

CRITICAL: API keys must NEVER be hardcoded in scripts. Always use .env files with restricted permissions. 
# Create .env file with restricted permissions
New-Item C:\AIWorkspace\.env -ItemType File
icacls C:\AIWorkspace\.env /inheritance:r
icacls C:\AIWorkspace\.env /grant AIAgent:R
icacls C:\AIWorkspace\.env /grant SYSTEM:F

# .env file should NOT be readable by other users
# .env file should be in .gitignore if using Git

Step 3.5 — Git Security

# Configure git to never commit secrets
# Create global gitignore
echo ".env" >> C:\Users\AIAgent\.gitignore_global
echo "*.key" >> C:\Users\AIAgent\.gitignore_global
echo "*.pem" >> C:\Users\AIAgent\.gitignore_global
echo "credentials.json" >> C:\Users\AIAgent\.gitignore_global
echo "token.json" >> C:\Users\AIAgent\.gitignore_global
git config --global core.excludesfile C:\Users\AIAgent\.gitignore_global

Security Checklist

  • Dedicated non-admin user account created
  • AI workspace folder with proper permissions
  • Windows Defender active with controlled folder access
  • Firewall allows only HTTPS outbound
  • .env file with restricted read permissions
  • Git configured to ignore secrets
  • Windows Update enabled (security patches)
  • BitLocker enabled on drive (if available)

4. Python Environment Setup

Step 4.1 — Install Python 3.11+

  1. Download from python.org/downloads
  2. IMPORTANT: Check "Add Python to PATH" during install
  3. Choose "Customize installation" > check all optional features
  4. Install for all users (if admin) or current user only

Step 4.2 — Verify Installation

python --version    # Should show 3.11+
pip --version       # Should show pip 23+

Step 4.3 — Create Virtual Environment

# Navigate to workspace
cd C:\AIWorkspace

# Create virtual environment
python -m venv ai-env

# Activate it
.\ai-env\Scripts\Activate.ps1

# Upgrade pip
python -m pip install --upgrade pip
Security: Always use a virtual environment. This isolates AI packages from the system Python and prevents dependency conflicts or supply-chain attacks from affecting the OS.

Step 4.4 — Install Core Packages

# Activate venv first!
.\ai-env\Scripts\Activate.ps1

# Core AI packages (all free/open-source)
pip install open-interpreter
pip install anthropic
pip install elevenlabs
pip install python-dotenv
pip install flask flask-cors
pip install requests

# Optional but useful
pip install rich          # Beautiful terminal output
pip install httpx         # Async HTTP client
pip install pydantic      # Data validation

Step 4.5 — Verify Packages

pip list | findstr "open-interpreter anthropic elevenlabs flask"
White-Label Note: Create a requirements.txt for each client deployment. Pin exact versions to ensure reproducibility: pip freeze > requirements.txt

6. Open Interpreter Setup

Open Interpreter lets the AI agent control the computer using natural language. It can run code, manage files, browse the web, and automate tasks — all from a chat interface.

What is Open Interpreter? It's like ChatGPT, but it can actually do things on your computer. It writes and runs code, manages files, installs packages, and controls applications. 55,000+ GitHub stars. Free. Open source.

Step 6.1 — Install (Already Done in Step 4.4)

# Should already be installed. Verify:
interpreter --version

Step 6.2 — First Run & Model Configuration

# Start Open Interpreter with Gemini (free)
interpreter --model gemini/gemini-2.0-flash

# Or with Claude (paid, better quality)
interpreter --model claude-sonnet-4-20250514

# Or with Groq (free, fastest)
interpreter --model groq/llama-3.3-70b-versatile

Step 6.3 — Create a Configuration Profile

Create C:\AIWorkspace\.interpreter\profiles\default.yaml:

# C:\AIWorkspace\.interpreter\profiles\default.yaml
llm:
  model: "gemini/gemini-2.0-flash"    # Free tier, good enough for most tasks
  api_key_env: "GOOGLE_API_KEY"        # Read from .env
  temperature: 0.3                     # Lower = more deterministic
  max_tokens: 4096

computer:
  language: "python"                   # Default execution language
  
safe_mode: "auto"                      # Ask before running dangerous commands
auto_run: false                        # Don't auto-execute code (safety first)
verbose: false                         # Keep output clean

Step 6.4 — Create the Lee AI Agent Profile

Create C:\AIWorkspace\.interpreter\profiles\lee-ai.yaml:

# Lee AI Agent Profile
llm:
  model: "gemini/gemini-2.0-flash"
  api_key_env: "GOOGLE_API_KEY"
  temperature: 0.3
  max_tokens: 4096
  context_window: 128000

system_message: |
  You are Lee AI, the digital assistant for Do Deals with Lee.
  You help manage the DDWL business by:
  - Managing files and documents in C:\AIWorkspace
  - Running Python scripts for data analysis
  - Generating reports and content
  - Automating repetitive tasks
  - Managing GHL CRM via API calls
  
  RULES:
  - Only work within C:\AIWorkspace directory
  - Never modify system files
  - Always confirm before deleting files
  - Log all actions to C:\AIWorkspace\logs\agent-log.txt
  - Never expose API keys in output

safe_mode: "auto"
auto_run: false
verbose: true

Step 6.5 — Run Lee AI

# Activate the virtual environment
cd C:\AIWorkspace
.\ai-env\Scripts\Activate.ps1

# Run with the Lee AI profile
interpreter --profile lee-ai

# You can now chat with Lee AI and it will execute tasks on the PC
# Example: "Create a folder called reports and generate a summary of today's deals"
Security: safe_mode: "auto" means Open Interpreter will ask for confirmation before running potentially dangerous commands (file deletion, network requests, system changes). Set to "off" only for fully trusted automated pipelines.

Step 6.6 — Create a Quick-Launch Script

Create C:\AIWorkspace\start-lee-ai.bat:

@echo off
echo ========================================
echo   LEE AI - Do Deals with Lee
echo   Powered by Open Interpreter
echo ========================================
cd C:\AIWorkspace
call ai-env\Scripts\activate.bat
interpreter --profile lee-ai
White-Label Note: For each client, create a custom profile YAML with their business name, system message, and rules. The .bat launcher can be branded with their company name. Place a shortcut on the desktop.

8. Windsurf Skills & Agent Configuration

Windsurf Skills are modular capability packages that teach Cascade (the AI) how to handle specific tasks. They load on demand, saving tokens.

Step 8.1 — Create Skills Directory

mkdir C:\AIWorkspace\.windsurf\skills\client-management

# Example: C:\AIWorkspace\.windsurf\skills\client-management\SKILL.md

name: client-management
description: Manage CRM operations, contacts, and workflows via GHL API.
---

## Connection Details
- API Key: Use GHL_API_KEY from .env
- Location ID: Use GHL_LOCATION_ID from .env

- Search contacts

- Send SMS/email

Create C:\AIWorkspace\AGENTS.md for project-wide instructions:

9. MCP Servers & Tool Connections

Model Context Protocol (MCP) is the universal connector for AI agents. Think of it as USB ports for AI — plug in any tool without code changes.

Step 9.1 — Configure MCP in Windsurf

Create or edit C:\AIWorkspace\.vscode\mcp.json:

{
    "servers": {
        "filesystem": {
            "command": "npx",
            "args": ["-y", "@anthropic/mcp-filesystem", "C:\\AIWorkspace"],
            "description": "Read/write files in the AI workspace"
        }
    }
}

Step 9.2 — Available MCP Servers (Free)

10. Agent Security & Permissions

The Security Model

ActionPermissionApproval
Read files in workspaceALLOWEDAuto
Write files in workspaceALLOWEDAuto
Run Python scriptsALLOWEDAuto
Make API calls (HTTPS)ALLOWEDAuto
Install pip packagesRESTRICTEDManual approval
Delete filesRESTRICTEDManual approval
Access outside workspaceBLOCKEDNever
Modify system filesBLOCKEDNever
Run as AdministratorBLOCKEDNever
Access other user accountsBLOCKEDNever

Open Interpreter Safety Settings

# In the profile YAML:
safe_mode: "auto"     # Ask before dangerous commands
auto_run: false       # Never auto-execute without review

# In AGENTS.md or system prompt:
# - Only work within C:\AIWorkspace
# - Never use 'rm -rf' or 'del /s /q' without confirmation
# - Never modify Windows registry
# - Never disable security software
# - Log all file modifications

Audit Logging

# Create a logging directory
mkdir C:\AIWorkspace\logs

# All agent actions should append to:
# C:\AIWorkspace\logs\agent-log.txt
# Format: [TIMESTAMP] [ACTION] [DETAILS]
# Example: [2026-02-15 11:30:00] [FILE_CREATE] Created reports/daily-summary.md
The balance: The agent CAN learn, create files, run scripts, call APIs, and do serious work. It CANNOT escape its workspace, modify the OS, or act without logging. This gives Lee AI the power to be productive while keeping the system safe.

14. Troubleshooting

ProblemCauseFix
Windsurf is very slow Too many extensions or low RAM Disable extensions, close other apps, increase page file
Open Interpreter won't start Virtual environment not activated Run .\ai-env\Scripts\Activate.ps1 first
"API key not found" .env file not loaded Check .env path, ensure load_dotenv() is called
"Rate limit exceeded" Too many API calls on free tier Switch to a different free provider (Groq, OpenRouter)
Agent tries to access system files System prompt not restrictive enough Update AGENTS.md and profile YAML with stricter rules
"pip install" fails Network or permissions issue Check firewall rules, ensure venv is activated
Cascade panel is blank Windsurf cache issue Ctrl+Shift+P > "Reload Window"
PC runs out of disk space Logs, temp files, pip cache Run pip cache purge, clear logs, run Disk Cleanup

Emergency Reset

# If everything is broken, reset the AI environment:
cd C:\AIWorkspace
rmdir /s /q ai-env
python -m venv ai-env
.\ai-env\Scripts\Activate.ps1
pip install -r requirements.txt
# All skills and configs are preserved (they're just .md files)
